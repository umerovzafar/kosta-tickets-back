# Подключение фронтенда к `tickets-back` (Kosta Legal)

Краткая инструкция для веб-клиента: **gateway** как единая точка входа, **JWT** в запросах, маршруты **time tracking** и новый список **партнёров**.

---

## 1. Базовый URL

Обычно в проде (пример): **`https://ticketsback.kostalegal.com/api/v1`**

Дальше все пути ниже **относительно** этого префикса, если у вас в коде `API_BASE` уже содержит `/api/v1`.

---

## 2. Авторизация

На каждый запрос (кроме health и явно публичных маршрутов):

```http
Authorization: Bearer <access_token>
```

Токен выдаёт **auth**-сервис. В браузере тот же токен нередко пробрасывается через **cookie** + **gateway** (см. настройки gateway: заголовок может подставляться из cookie сессии).

---

## 3. Префикс модуля «учёт времени» (через gateway)

Путь к API учёта времени:

```text
{API_BASE}/time-tracking/...
```

Пример: список пользователей TT — `GET {API_BASE}/time-tracking/users`.

---

## 4. Маршруты, важные для сценария «доступ к проектам»

| Метод | Путь (после `/time-tracking/`) | Назначение |
|--------|----------------------------------|------------|
| `GET` | `users` | Справочник пользователей TT (как в ответе — зависит от роли) |
| `GET` | **`users/partners`** | **Только** пользователи, подходящие под роль/должность **партнёра** (должность в TT или в auth, плюс орг-роль из auth) — для выбора на UI |
| `PUT` | `users/{authUserId}/project-access` | Сохранение списка `projectIds` (доступ к списанию времени) |
| `POST` | `clients/{clientId}/projects` | Создание проекта; тело может включать `initialTimeTrackingUserAuthIds` |
| `PATCH` | `clients/{clientId}/projects/{projectId}` | Правка проекта |

**Поведение `GET users/partners` на gateway (как и у `GET users`):** администраторам/офису отдаётся полный список партнёров; **менеджеру** — вариант **ограниченный зоной** (тот же принцип, что и `users` / `managed-scope` на бэке).

---

## 5. Правило «на проекте должен быть партнёр»

Бэкенд не полагается только на поле `position` в БД TT. При сохранении доступа передавайте тот же **`Authorization`**, иначе при пустом `position` в TT, но заполненном **профиле auth**, проверка могла бы не увидеть партнёра.

Считаются партнёрами, если:

- в **должности** (TT или auth) есть «партн…» / `partner` (без учёта регистра), **или**
- **орг-роль** в auth (поле `role` в ответе `/users` auth) указывает на партнёра по тем же подстрокам.

Подробности по сопутствующим полям и 400/ошибкам:  
`time_tracking/docs/FRONTEND_PROJECT_ACCESS_PARTNER.md` и `docs/FRONTEND.md`.

---

## 6. Окружения и CORS

- Убедитесь, что **origin** фронта (например `https://tickets.kostalegal.com`) разрешён для **CORS** на gateway (и при необходимости на отдельных сервисах).
- Между `http://localhost` и прод-доменами путь и куки могут отличаться — сверяйте с настройками стенда.
