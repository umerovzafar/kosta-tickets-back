Сброс данных учёта времени (time tracking)

Выход

Навигация

Учёт времени — полный сброс бизнес-данных

Удаляются клиенты, проекты, задачи и категории по клиентам, записи времени, доступ к проектам, почасовые ставки в БД учёта времени, счета (инвойсы), сохранённые отчёты и снимки. Пользователи раздела «Учёт времени» (таблица time_tracking_users) не удаляются. Пользователи в auth / общий список в админке не затрагиваются этим действием.

Кому доступно

Только роль «Главный администратор».

Опасная зона

  • Действие необратимо. При необходимости сделайте резервную копию БД kosta_time_tracking.
  • В Docker из этого репозитория по умолчанию сброс разрешён (см. docker-compose.yml). Если видите ошибку 409 — в Portainer/.env для сервиса time_tracking задайте TIME_TRACKING_ALLOW_BUSINESS_DATA_RESET=true или уберите строку со значением false, затем перезапустите контейнер time_tracking.

Подтверждение в диалоге и фраза RESET_TIME_TRACKING_BUSINESS_DATA.
