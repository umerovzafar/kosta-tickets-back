# Фронтенд: неоплачиваемые часы (non-billable)

Бэкенд time tracking уже:

- хранит флаг **`is_billable`** у записей времени;
- поддерживает **`task_id`** у записи: задача из справочника клиента (**`GET /clients/{id}/tasks`**) задаёт тип **оплачиваемая / неоплачиваемая** через **`billable_by_default`**; при указании **`task_id`** сервер выставляет **`is_billable`** в соответствии с задачей;
- в **`GET /clients/{client_id}/projects/{project_id}/dashboard`** отдаёт **`tasks`**: часы по каждой задаче (и строки «Без задачи (оплачиваемые|неоплачиваемые)» для записей без **`task_id`**), плюс **`totals`**, **`hours_by_week`**, **`team`**.

Ниже — что проверить и доработать на фронте (репозиторий `tickets-front`).

## Уже в порядке (проверить после деплоя API)

1. **`normalizeProjectDashboard`** (`src/entities/time-tracking/api.ts`) — маппинг **`non_billable_hours` → `nonBillableHours`** уже есть; после исправления бэкенда карточки/цифры на деталях проекта, которые читают **`dashboard.totals.nonBillableHours`**, должны заполняться сами.
2. **Табель / форма записи** (`TimesheetPanel.tsx`):
   - добавить в тип записи и в тело create/patch поле **`task_id`** (в gateway — **`taskId`**), после выбора проекта подгружать список задач клиента и давать выбрать задачу;
   - если выбрана задача, **`isBillable`** на бэкенде берётся из задачи — на UI либо блокировать переключатель, либо синхронизировать его с **`billable_by_default`** выбранной задачи;
   - без задачи по-прежнему можно вручную задавать **`isBillable`**.

## Обязательные доработки (если поведение всё ещё «как будто нет» неоплачиваемых)

1. **Загрузка дашборда** (`ProjectDetailPage.tsx`): сейчас вызывается **`getClientProjectDashboard(clientId, projectId)`** без **`dateFrom` / `dateTo`**. Это соответствует «вся история проекта» на бэкенде. Если нужен дашборд **за тот же период**, что и вкладка «Команда» / отчёты — передавать в **`getClientProjectDashboard`** третий аргумент **`{ dateFrom, dateTo }`** в формате **`YYYY-MM-DD`**, синхронизированный с выбранным периодом в UI.
2. **Блок задач на странице проекта** (`taskData` в `ProjectDetailPage.tsx`): бэкенд теперь заполняет **`dashboard.tasks`**. Убедиться, что при **`tasks.length > 0`** не подменяются данные заглушкой **`genTasks`**. Строки с **`task_id`** вида **`__unassigned_billable__` / `__unassigned_non_billable__`** — агрегат часов без привязки к задаче; их можно показывать как есть или стилизовать отдельно.
3. **Ошибка дашборда**: при **`getClientProjectDashboard` → `catch`** выставляется **`dashboard = null`**, и UI уходит в **фолбэки с фиктивными пропорциями** (`genTasks`, расчёт часов от `spent`). Нужно различать «нет эндпоинта / 404» и «ошибка сети» и не подменять реальные нули вымышленными неоплачиваемыми часами.

## Отчёты (`ReportsPanel.tsx`)

- Сводка по времени сейчас **заглушка**: **`nonBillableHours = 0`**, **`billableHours = 0`** и т.д.
- Чтобы в отчётах отображались неоплачиваемые часы, нужен **отдельный контракт API** (агрегаты за период по пользователям/проектам/задачам) и подключение его в панель отчётов; текущего дашборда проекта для глобальных отчётов недостаточно.

## Опционально (UX)

1. **График часов по неделям** (`buildChartDataFromDashboard`): сейчас в значение попадает только **`hours`** (сумма). Бэкенд отдаёт ещё **`billable_hours`** и **`non_billable_hours`** по неделям — можно сделать **stacked bar** или вторую серию для неоплачиваемых.
2. **Страница загрузки команды** уже использует **`non_billable_hours`** из team workload API — при расхождении с дашбордом проекта проверить, что периоды и фильтры совпадают.
